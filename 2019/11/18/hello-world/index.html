    踩過的那些坑

    hexo deploy 報錯

    當時遇到錯誤的訊息時，忘記截圖好做成筆記了，總之我也是上網爬了一下文，這邊借用一下別人的錯誤訊息，
    大致上是跟我遇到的問題一樣。

    解決方法：跟著官方文件的步驟做，基本上就可以解決摟 ～

    Could not resolve hostname github.com:zhengpq: Name or service not known

    解決方法： 把deply repo的值換成https的Git倉庫clone地址就可以了
    tip: ( 記得如果你本來是使用ssh 遠程倉庫的 URL ，要更新新使用 HTTPS)

    • git remote set-url 命令可更改現有遠端資料庫的 URL

    Example: 將遠端 URL 從 SSH 切換到 HTTPS

    Ｍac 權限

    當我排除萬難設置好文件，打了指令 hexo d 準備期待發佈我的網站時，又出現了錯誤訊息…

    其實這個也是權限的問題，我最後是在指令的前面加上 sudo

    補充: sudo 是 Unix/Linux 平臺上的一個非常有用的工具，它允許系統管理員分配給一般用戶合理的權限，讓他們執行一些只有系統管理員或其他特定帳號才能完成的任務，例如：執行 mount, halt, su 這類的指令，或者編輯系統設定檔 (/etc/mtab, /etc/samba/smb.conf 等)。這樣一來，不僅減少了 root 使用者的登錄次數和管理時間，也提高了系統安全性。)
